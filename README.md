## Git Commit Message Types (Conventional Commits)

Format:

```
<type>: <short description>
```

---

### feat
New feature

```
feat: add user authentication
feat: implement dark mode toggle
```

### fix
Bug fix

```
fix: resolve login validation error
fix: fix broken navigation link
```

### docs
Documentation changes

```
docs: update README installation steps
docs: add API usage examples
```

### style
Code style changes (formatting, spaces, etc.)

```
style: format code with prettier
style: fix indentation in main file
```

### refactor
Code restructuring without changing functionality

```
refactor: simplify authentication logic
refactor: extract helper functions
```

### perf
Performance improvements

```
perf: optimize database queries
perf: reduce page load time
```

### test
Adding or updating tests

```
test: add unit tests for login module
test: improve API test coverage
```

### build
Build system or dependency changes

```
build: update webpack configuration
build: upgrade project dependencies
```

### ci
CI/CD configuration changes

```
ci: add GitHub Actions workflow
ci: update deployment pipeline
```

### chore
Maintenance tasks

```
chore: update dependencies
chore: remove unused files
```

### revert
Reverting previous commits

```
revert: revert login feature commit
revert: undo previous API changes
```

### wip
Work in progress

```
wip: working on dashboard layout
wip: implementing authentication system
```
